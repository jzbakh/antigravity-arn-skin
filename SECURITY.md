# Security Policy

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in ARN-Skin,
please report it **privately** through GitHub's built-in
[private vulnerability reporting](https://github.com/jzbakh/antigravity-arn-skin/security/advisories/new)
feature on this repository.

Reports are end-to-end confidential — only the maintainer
([@jzbakh](https://github.com/jzbakh)) can read your submission. Do **not**
open a public issue, post in Discussions, or share details on social
media until the vulnerability has been addressed and a patch is
available.

## What to include

A useful report typically contains:

- A clear description of the vulnerability and the conditions that
  trigger it.
- Steps to reproduce, ideally with the smallest possible test case.
- The version of ARN-Skin, the editor (VSCode / Cursor / Antigravity /
  VSCodium / Windsurf), and the operating system on which the issue was
  observed.
- Any potential impact you have identified.
- *(Optional)* a suggested fix, if you have one.

## What to expect

- **Acknowledgement** within 7 days of your report.
- **Initial assessment** within 14 days, with a preliminary plan and
  timeline.
- **Coordinated disclosure** once a fix is available — you will be
  credited in the release notes if you wish.

## Scope

The supported version is the **latest published release** on the
[VSCode Marketplace](https://marketplace.visualstudio.com/items?itemName=jzbakh.antigravity-arn-skin)
and [Open VSX](https://open-vsx.org/extension/jzbakh/antigravity-arn-skin).
Earlier versions do not receive security updates — please upgrade to the
latest release before reporting.

## Out of scope

The following are **not** considered security vulnerabilities for this
project:

- Visual or aesthetic concerns about colour palettes.
- Compatibility issues with non-VSCode-based editors.
- Issues caused by other extensions interfering with ARN-Skin.
- Use of the extension in unsupported configurations *(modified VSIX,
  manual installation outside the standard extension directories, etc.)*.

For non-security bug reports, please use the
[Bug report issue template](https://github.com/jzbakh/antigravity-arn-skin/issues/new?template=bug_report.md)
instead.
